# PetAgent 欢迎页 Logo 替换 — 踩坑记录

> 目标：将 Hermes Agent v0.17.0 欢迎页的大标题 HERMES AGENT 替换为自定义像素猫 Logo + 副标题「有什么可以帮你的喵！~」
> 总迭代次数：5 次
> 最终解决方案：v2.0.19 — 纯 CSS 伪元素覆盖

---

## 一、问题总览

### 技术背景
- **Hermes Agent v0.17.0** 使用 React 前端框架，构建产物为 `dist/assets/index-*.js`
- 欢迎页组件路径：`src/components/chat/intro.tsx`
- 运行时加载的是 **编译后的 JS**（`dist/assets/index-*.js`），而不是源码
- React 的虚拟 DOM 机制：JS 运行时不断调用 `createElement`/`jsx`，重新渲染时会覆盖任何手动注入的 DOM 节点

### 关键发现
编译后的 `dist/assets/index-*.js` 中**不存在** `.intro-petagent` 或 `.intro-hermes` 这些 className，这意味着源码 `intro.tsx` 中即使有这些 className，编译后也被 React 打包成了其他形式或消失了。所有依赖这两个 class 的 CSS 规则和 DOM 操作都不可能生效。

## 二、踩坑历程

### 尝试 1：v2.0.14 → 2.0.15 — 用 set-html 替换整个欢迎页容器

**方案：**
- `customizations.json`：对 `welcomeContainer`（锚点 `[data-slot="aui_intro"]`）执行 `set-html`，注入 `<img src="./pet-agent-logo.png">`
- CSS 切换规则写在 `custom-theme.css` 中（.intro-hermes / .intro-petagent 显示隐藏）

**失败原因：**
- 安装器只把 `petagent-theme.css` 复制到 dist/，而 CSS 规则写在了 `custom-theme.css` 中——后者根本不部署
- `set-html` 虽然执行了，但 React 很快重新渲染了欢迎页，覆盖了我们注入的 HTML

### 尝试 2：v2.0.15 — 发现 Hermes v0.17 内置了 .intro-petagent div

**方案：**
- 检查 Hermes v0.17 源码 `intro.tsx`，发现它内置了 `.intro-hermes` 和 `.intro-petagent` 结构
- 修改 `customizations.json`：用 `replace-children` 替换 `.intro-petagent` 的内容，用 `set-text` 设置副标题
- 恢复 CSS 切换规则到 `petagent-theme.css`

**失败原因：**
- 源码中的 `.intro-petagent` 和 `.intro-hermes` class 名在编译后**不存在**于 `dist/assets/index-*.js` 中
- 引擎用 `document.querySelector('.intro-petagent')` 永远找不到元素，所以不会执行任何操作
- CSS 规则 `.intro-hermes { display: none; }` 也没用，因为选择器匹配不到任何元素

### 尝试 3：v2.0.16 — 修复 CSS 覆盖逻辑和文件覆盖问题

**方案：**
- 在 `petagent-theme.css` 中添加 intro 切换规则（.intro-hermes / .intro-petagent）
- 修复 `installer.py`：更新时不再跳过已存在的文件（`if dst.exists(): continue`）
- 升级版本到 2.0.16，重新构建三个产品

**失败原因：**
- 虽然 CSS 规则正确写入了 `petagent-theme.css`（已验证部署后文件中有规则），但 `.intro-petagent` 和 `.intro-hermes` 在编译后的 JS 中**根本不存在**
- 这意味着即使 CSS 规则正确加载，也永远不会匹配到 DOM 元素

### 尝试 4：v2.0.18 — 用 `welcomeContainer` + `set-html` + `persistent: true`

**方案：**
- 直接定位 `[data-slot="aui_intro"]`（编译后 JS 中真实存在的属性）
- `action: set-html` 替换整个欢迎页内容
- `persistent: true` 让引擎在 React 重新渲染后再次注入

**失败原因：**
- React 的渲染周期非常快，欢迎页在初始加载时会被渲染一次，之后对话开始后还会消失（切换到聊天视图）
- 虽然 `persistent: true` 会重新注入，但 React 的渲染时机和 MutationObserver 的触发时机不同步，导致用户看到的最终画面仍然是原始内容
- 根本原因是：React 在内存中维护虚拟 DOM，然后 diff 后批量写入真实 DOM，任何外部修改都可能被 React 的 diff 算法当作"需要恢复为原始状态"而覆盖

### 尝试 5：v2.0.19 — 纯 CSS 伪元素覆盖（✅ 成功方案）

**方案：**
- **放弃 DOM 操作**：彻底移除 `customizations.json` 中的欢迎页 `dom.customizations` 项（只保留侧边栏品牌）
- **纯 CSS 伪元素**：用 `[data-slot="aui_intro"]::before` 显示背景图（像素猫 logo），用 `::after` 显示副标题文字
- **隐藏原始内容**：`[data-slot="aui_intro"] > div { display: none !important; }` 隐藏 React 渲染的原始 HERMES AGENT 文字

**为什么成功：**
- CSS 规则不会被 React 覆盖（React 的 diff 算法只比较 DOM 结构和 props，不修改 CSS 样式表）
- `[data-slot="aui_intro"]` 是编译后 JS 中真实存在的数据属性，始终存在
- `::before` / `::after` 伪元素是 CSS 层面的内容，React 的虚拟 DOM 检测不到它们，也不会试图删除它们
- 暗色模式用 `filter: invert(1)` 处理黑色像素猫在暗色背景下的可见性

## 三、核心经验教训

### 教训 1：React 编译产物 ≠ 源码
永远不要假设源码中的 className 会在编译后 JS 中保留。Webpack/Vite 的 tree shaking、CSS 模块化处理、以及 React 的 JSX 编译，都可能导致源码中的字符串在编译后变成完全不同的形式。唯一可靠的验证方法是：直接搜索 `dist/assets/index-*.js` 中的字符串。

### 教训 2：React 会覆盖 DOM 操作，但不会覆盖 CSS
对 React 管理的组件进行 `innerHTML = ...` 或 `appendChild` 等 DOM 操作，结果只能是昙花一现——React 的虚拟 DOM diff 算法会在下一次渲染时恢复为 React 状态树中的原始内容。

唯一可靠的方法是：
- **CSS 样式覆盖**：React 不会修改外部 CSS 样式表中的规则
- **伪元素（::before / ::after）**：React 的 diff 算法看不到伪元素，不会尝试删除它们
- **CSS 变量覆盖**：React 组件可能使用 CSS 变量，我们可以覆盖这些变量

### 教训 3：数据属性（data-slot）是编译后的可靠锚点
Hermes 的编译产物保留了 `data-slot` 属性（如 `data-slot="aui_intro"`、`data-slot="sidebar-content"`）。这些属性在编译后 JS 中作为字符串字面量存在，不会被 tree shaking 掉，是可靠的 CSS 选择器锚点。

### 教训 4：CSS 文件必须写入实际部署的文件
本项目有 `custom-theme.css` 和 `petagent-theme.css` 两个文件。`custom-theme.css` 只是开发时的参考文件，**安装器只部署 `petagent-theme.css`**。任何 CSS 规则必须写入 `petagent-theme.css` 才会被部署到 Hermes 的 dist/ 目录中。

### 教训 5：更新时旧文件不会被覆盖
`installer.py` 的原始逻辑对 `copy` 操作有 `if dst.exists(): continue`，这意味着已安装的 logo 和 CSS 文件永远不会被新版本覆盖。必须使用 `shutil.copy2()` 强制覆盖，否则更新后文件内容仍然是旧版本。

## 四、关键文件清单

| 文件 | 作用 | 踩坑要点 |
|------|------|----------|
| `src/engine/customizations.json` | DOM 操作配置清单 | 欢迎页不要在这里配 DOM 操作，React 会覆盖 |
| `src/modules/theme/copy/apps/desktop/src/petagent-theme.css` | 实际部署的 CSS 文件 | 唯一可靠的地方写 CSS 规则；`custom-theme.css` 不部署 |
| `src/modules/theme/copy/apps/desktop/src/custom-theme.css` | 开发参考 CSS 文件 | **不实际部署**，只是给人看的 |
| `src/modules/pet/copy/apps/desktop/public/pet-agent-logo.png` | 像素猫 logo 图片 | 透明背景 PNG，在暗色模式下需要 `filter: invert(1)` |
| `src/installer.py` | 安装器核心逻辑 | 必须覆盖 `if dst.exists(): continue` 才能正确更新文件 |
| `src/engine/adapters/hermes-v0.16.js` | 语义锚点适配器 | 锚点选择器必须匹配编译后 JS 中的真实属性 |
| `src/engine/petagent-engine.js` | 浏览器端运行时引擎 | `set-html` 和 `replace-children` 在 React 环境下不可靠 |
| `src/electron-app/panel.js` | 模块管理器 UI | 加了「打开 Hermes」按钮，方便更新后直接启动 |

## 五、最终正确的 CSS 方案（v2.0.19）

```css
/* 隐藏 React 渲染的原始内容 */
[data-hermes-theme="petagent"] [data-slot="aui_intro"] > div {
  display: none !important;
}

/* 用 ::before 伪元素显示像素猫 logo */
[data-hermes-theme="petagent"] [data-slot="aui_intro"]::before {
  content: '';
  display: block;
  height: 100px;
  width: 28rem;
  max-width: 90%;
  background-image: url('./pet-agent-logo.png');
  background-size: contain;
  background-repeat: no-repeat;
  background-position: center;
  margin-bottom: 0.75rem;
  flex-shrink: 0;
}

/* 用 ::after 伪元素显示副标题 */
[data-hermes-theme="petagent"] [data-slot="aui_intro"]::after {
  content: '有什么可以帮你的喵！~';
  display: block;
  text-align: center;
  line-height: 1.5;
  letter-spacing: -0.01em;
  font-size: 1.05rem;
  color: var(--dt-muted-foreground, #6B6B70);
}

/* 暗色模式：黑色 logo 在暗色背景上反色 */
[data-hermes-theme="petagent"].dark [data-slot="aui_intro"]::before {
  filter: invert(1) hue-rotate(180deg);
  opacity: 0.9;
}
```

## 六、验证方法

1. 部署后打开 Hermes，按 F12 打开开发者工具
2. 检查 `[data-slot="aui_intro"]` 元素：
   - 其子 `div` 应该被 `display: none !important` 隐藏
   - 应该能看到 `::before`（像素猫 logo）和 `::after`（副标题）
3. 如果看不到，检查 `petagent-theme.css` 是否被 `@import` 到 CSS bundle 中
4. 检查 `<html>` 是否有 `data-hermes-theme="petagent"`（localStorage 预注入设置）
